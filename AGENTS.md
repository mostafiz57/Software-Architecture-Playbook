# Agent Instructions

You are writing professional Architecture Design Documents (ADD) for the Software Architecture Playbook.

## Rules

- Write one section at a time. Stop after Section 1 and wait for user review.
- Follow the 19-section ADD structure exactly.
- Use Markdown tables, Mermaid diagrams, and code examples where appropriate.
- Explain every architectural decision with reasoning, trade-offs, and alternatives.
- Do not write interview-style answers. Write as if documenting a real enterprise system.
- Use Clean Architecture, DDD, SOLID, Microservices, Event-Driven Architecture, CQRS, Cloud-Native Design, Scalability, High Availability, Security by Design, Observability, and Maintainability principles where appropriate.
- Do not force patterns into every design. Explain when simpler approaches are better.
- Avoid unnecessary theory. Be practical and production-oriented.
- Target audience: Software Engineers, Senior Developers, Technical Leads, Solution Architects, Engineering Managers, and Students.

## File Naming

Save case studies as `case-studies/<system-name>/README.md`.

Example: `case-studies/e-commerce-platform/README.md`

## Workflow

1. Create the case study directory under `case-studies/`.
2. Write Section 1: Introduction.
3. Wait for user feedback.
4. Write Section 2, and so on.
